Hinweise

Folgende Hinweise sollen ihnen helfen, RPGUnit richtig anzuwenden und möglicherweise unerwartetes Verhalten richtig zu interpretieren.

Prinzipielle Arbeitsweise

Grundsätzlich bemüht sich RPGUnit die Arbeitsweise von JUnit exakt nachzubilden. Problematisch ist hierbei jedoch, wenn Zusicherungen überwacht und Fehler somit abgefangen werden. Ist hierbei der Parameter ONFAILURE auf *CONTINUE gesetzt, erkennt RPGUnit die Überwachung nicht und meldet einen Fehler.

Das Problem existiert nicht, wenn ONFAILURE auf *ABORT gesetzt ist, welches dem Standardwert entspricht.

Monitor Block 

monitor;

   // calculation statements
   
   aEqual( 'HelloWorld' : 'Hello' );  

on-error;
   // error handler
endmon;

CALLP - Error Extender

callp(e) aEqual( 'HelloWorld' : 'Hello' );
